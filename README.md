# CustomATS

An AI-powered Applicant Tracking System that uses a Prolog reasoning engine to rank candidates against job requirements, then summarizes results in natural language via a local Ollama model.

---

## Prerequisites

Before running the project, make sure you have the following installed:

- **Node.js** v18 or higher — [https://nodejs.org](https://nodejs.org)
- **Ollama** — [https://ollama.com](https://ollama.com)

---

## 1. Install Ollama and pull a model

### Install Ollama

Download and install Ollama for your OS from [https://ollama.com/download](https://ollama.com/download).

### Pull the model

The project is currently configured to use `qwen3:8b`. Pull it before running:

```bash
ollama pull qwen3:8b
```

### Start Ollama

Ollama must be running locally when you use the app. Start it with:

```bash
ollama serve
```

> Ollama listens on `http://localhost:11434` by default. If you see a "server already running" message, it is already up.

---

## 2. Change the Ollama model (optional)

To use a different model, open **`config.json`** in the project root and update the `ollamaModel` field:

```json
{
  "cvsFolder": "data/candidate_cvs",
  "ollamaModel": "qwen3:8b"
}
```

Replace `"qwen3:8b"` with any model you have pulled locally (e.g. `"llama3.2"`, `"mistral"`, `"gemma3"`). Make sure you run `ollama pull <model-name>` first.

---

## 3. Install Node dependencies

From the project root:

```bash
npm install
```

---

## 4. Run the app

### Development mode (auto-reloads on file changes)

```bash
npm run dev
```

### Production mode

```bash
npm start
```

The server starts on **http://localhost:3000**. Open that URL in your browser.

---

## 5. Using the app

1. The main page shows a list of IT skill categories. Use the search box or expand the accordions to find skills.
2. Check the skills required for the position you are hiring for.
3. Click **Submit**. The button will show a spinner while the system processes the query.
4. A modal will open with a natural-language summary of the top 3 candidates generated by Ollama.

> If Ollama is not running, the modal will fall back to showing the raw ranked output instead.

---

## 6. Run the tests

```bash
npm test
```

To run a single test file:

```bash
npm test -- tau.test.js
npm test -- candidate-facts.test.js
```

To run tests matching a name pattern:

```bash
npm test -- --testNamePattern="exact_match"
```

## Project structure

```
index.js                  Express server (routes: /, /categories, /query, /extract-candidates)
config.json               Runtime config — set ollamaModel here
prolog/
  base.pl                 Skill atoms + transferability rules (~1200 lines)
  tail.pl                 Scoring predicates (evaluate_match, top_applicants_breakdown)
public/
  index.html              Main UI (Bootstrap 5)
  style.css               Styles
  scripts/
    main.js               Form logic, submit handler, result rendering
    tau.js                Async tau-prolog wrapper
    candidate-facts.js    Converts candidate JSON to Prolog has_skill/2 facts
data/
  form.json               IT skill taxonomy (source of truth for the UI checkboxes)
  candidate_cvs/          Production candidate JSON files
  test_cvs/               Candidate files used by integration tests
test/integration/         Jest integration tests
```
